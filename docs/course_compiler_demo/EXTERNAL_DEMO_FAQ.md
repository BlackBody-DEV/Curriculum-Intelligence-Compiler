# External Demo FAQ

## What does this compile?

The current MVP compiles a local sample Algebra I source document into candidate topics, candidate micro-skills, demo practice items, demo assessment items, a performance tracking preview, a content gap report, a demo report, and a validation report.

## Is this using AI?

The current MVP is a local demo pipeline with rule-based extraction and packaging logic. It is best described as a prototype compiler slice, not a production AI curriculum system.

## Is it production-ready?

No. This is not production-ready content. All learning outputs are `demo_unverified`, and the generated practice, assessment, and tracking artifacts require review before any classroom or platform use.

## Can it handle non-math subjects?

Not as a supported demo claim. Multi-subject expansion is post-MVP work. The roadmap recommends a carefully isolated physics demo slice after the math lane remains stable.

## Does it connect to the AxiomIQ Alpha platform?

No. No operational Alpha integration is present. The MVP does not update operational Alpha readiness or mastery.

## Does it use student data?

No. The demo uses a local sample input file and does not contact any database, student data store, hosted service, or live API.

## Can teachers edit the generated content?

Not in the current MVP workflow. Future product work could add review and editing workflows, but the current outputs are demo artifacts and should remain `demo_unverified`.

## What remains before classroom use?

Classroom use would require reviewed source handling, stronger schemas, human content review, approval workflows, item-bank quality checks, privacy and rights review, production validation, and a separate integration plan.
