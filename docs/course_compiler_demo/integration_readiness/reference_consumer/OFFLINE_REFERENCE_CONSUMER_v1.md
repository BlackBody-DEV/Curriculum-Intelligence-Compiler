# Offline Reference Consumer v1

The offline reference consumer proves that a portable compiler release package can be loaded, semantically validated, integrity checked, internally resolved, enumerated, classified, and converted into deterministic dry-run planning evidence.

It is not an importer. It does not read or write adaptive-platform, contact a database, call runtime routes, mutate learner state, promote canonical content, or create student-visible output.

## Processing Sequence

1. package-root safety check
2. package discovery
3. semantic validator invocation
4. validation-verdict gate
5. manifest loading
6. declared-artifact enumeration
7. internal-reference resolution
8. dependency ordering
9. artifact classification
10. dry-run integration-plan generation
11. compatibility-warning aggregation
12. boundary-evidence generation
13. deterministic verdict aggregation

## Validator Dependency

The consumer depends on the frozen `compiler_release_package_semantic_validator` version `1.0.0`. Validator `accept` and `accept_with_warnings` verdicts may proceed. Validator `reject` returns `rejected_without_plan`. Validator errors return `consumer_error`.

## Non-Mutation Guarantees

The consumer reads package files without repair, rewrite, normalization, regeneration, chmod, rename, delete, or cache creation. Report-writing mode writes only compiler-owned report artifacts below the requested safe output directory.

## Future Lane F Handoff

Compatibility conclusions about an actual destination remain out of scope. Lane F owns platform compatibility findings. This consumer records only generic future capability requirements.
